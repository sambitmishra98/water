## Purpose

`Efforts/profession/ExampleProject` is the active execution hub for this
project's hands-on work (e.g. running simulations, code, or experiments on a
shared cluster or lab machine).

Execution workspace (illustrative): `/scratch/EFFORTS/ExampleProject/`.

## Rules

- Keep environments, dependencies, and reusable software setup outside the
  vault, in the actual working directory.
- Track commands, configs, failure modes, outputs, and next actions.
- Use a dedicated note for tracking your changes relative to any externally
  received baseline code.
- Do not paste full code archives into effort notes.
- Keep heavy outputs in Supplementary, Drive, or an external archive.
