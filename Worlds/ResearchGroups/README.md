## Purpose

`Worlds/ResearchGroups` stores research-group-facing child vaults and shareable lab knowledge interfaces.

It is for group-specific communication, onboarding, proposals, paper context, validation context, and curated project knowledge.
These are miniature interface vaults that should eventually be able to stand alone.
They may have their own `.obsidian` when ready.
Use Google Drive for sync and sharing when large data is involved.

## Inheritance

This README inherits `[[Worlds/README]]` and the root vault contract in `[[README]]`.

The sections below add local rules for research-group interfaces.

## What belongs here

- Research-group child vaults.
- Group-specific dashboards and maps.
- Proposal, paper, validation, onboarding, and project context.
- Curated summaries that can eventually be shared with collaborators.

## What does not belong here

- Private parent-vault material unless explicitly curated.
- Raw private memories or heavy archives.
- Unreviewed drafts that imply commitments or team positions.

## Local conventions

- Treat each group folder as potentially shareable.
- Prefer self-contained summaries with safe source links.
- Use Google Drive packet manifests for large payloads.
- Curate parent-vault private context before inclusion.
